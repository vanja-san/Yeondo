# Yeondo — Утилита для создания символических ссылок

Простое и удобное приложение для массового создания символических ссылок в Windows.

![Версия](https://img.shields.io/badge/версия-1.10.5-blue)
![.NET](https://img.shields.io/badge/.NET-10-purple)
![Платформа](https://img.shields.io/badge/платформа-Windows%20x64-lightgrey)

## 📌 Возможности

- **Три типа ссылок:** Symbolic Link, Junction, Hard Link
- **Массовое создание:** Добавляйте файлы и папки пакетами
- **Drag & Drop:** Перетаскивайте файлы прямо в окно приложения
- **Автоматическое определение языка:** Русский или английский интерфейс
- **Компактный интерфейс:** Современный дизайн в стиле Windows 11
- **Логирование:** Подробный отчёт о результатах создания ссылок

## 🚀 Быстрый старт

### 1. Установка

Скачайте и распакуйте архив с приложением в любую папку.

**Требования:**
- Windows 10/11 x64
- .NET 10 Desktop Runtime *(если используете версию без встроенного runtime)*

### 2. Запуск

Запустите файл `Yeondo.exe`

### 3. Создание ссылки

1. **Добавьте файлы или папки:**
   - Нажмите кнопку 📄 (файлы) или 📁 (папки)
   - Или перетащите файлы в окно приложения

2. **Выберите тип ссылки:**
   - **Symbolic** — универсальные ссылки (файлы и папки)
   - **Junction** — только для папок
   - **Hard Link** — только для файлов

3. **Укажите целевую папку:**
   - Нажмите «Обзор» и выберите папку назначения
   - Или кликните на путь, чтобы открыть папку в проводнике

4. **Нажмите «Создать»**

## 📖 Типы ссылок

| Тип | Для чего | Особенности |
|-----|----------|-------------|
| **Symbolic** | Файлы и папки | Работает как ярлык, требует прав администратора (без привилегий разработчика) |
| **Junction** | Только папки | Работает на уровне файловой системы, не требует прав |
| **Hard Link** | Только файлы | Файл должен существовать, работает только в пределах одного тома NTFS |

## ⌨️ Горячие клавиши

| Действие | Клавиши |
|----------|---------|
| Добавить файлы | — |
| Добавить папки | — |
| Создать ссылки | Enter (когда кнопка активна) |
| Открыть контекстное меню | Правая кнопка мыши на элементе |

## 🌐 Локализация

Приложение автоматически определяет язык системы:

- **Русский** — если язык системы русский
- **English** — для всех остальных языков

Файлы локализации хранятся в папке `i18n/` рядом с исполняемым файлом:

- `ru.json` — русский язык
- `en.json` — английский язык

Вы можете отредактировать эти файлы для изменения текстов интерфейса.

## ❓ Устранение неполадок

### Ошибка при создании Hard Link

**Причина:** Файл находится на другом томе или диске.

**Решение:** Hard Link работает только в пределах одного раздела NTFS.

---

### Ошибка при создании Junction

**Причина:** Выбран файл вместо папки.

**Решение:** Junction работает только с папками.

---

### Ошибка «Отказано в доступе»

**Причина:** Недостаточно прав для создания символических ссылок.

**Решение:** Запустите приложение от имени администратора или включите «Режим разработчика» в Windows 10/11.

---

### Приложение не запускается

**Причина:** Не установлен .NET 10 Runtime.

**Решение:** Скачайте и установите [.NET 10 Desktop Runtime](https://dotnet.microsoft.com/download/dotnet/10.0)

## 📝 Логирование

Файлы логов создаются в папке с приложением:
```
./logs/symlink_ГГГГММДД_ЧЧММСС.log
```

Для просмотра логов нажмите кнопку **«Логи»** в статусной строке (появляется при ошибках).

**Все файлы приложения (настройки, локализация, логи) создаются рядом с исполняемым файлом** — никаких системных папок!
